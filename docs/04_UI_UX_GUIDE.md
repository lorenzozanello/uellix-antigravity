# 04 — Guía UI/UX

## Personalidad visual

Uellix debe sentirse:

- riguroso;
- premium;
- tecnológico;
- institucional;
- confiable;
- claro;
- sobrio;
- moderno.

No debe sentirse:
- infantil;
- excesivamente decorativo;
- críptico;
- académico pesado;
- genérico;
- tipo plantilla barata de dashboard.

## Tono de experiencia

La plataforma debe acompañar al usuario en un proceso metodológico complejo sin abrumarlo.

El usuario debe sentir:

> “Estoy construyendo un análisis serio y defendible, con guía experta paso a paso.”

## Principios UX

1. Cada pantalla debe tener una acción principal.
2. Cada paso del SROI Pipeline debe explicar:
   - qué se hace;
   - por qué importa;
   - cómo completarlo;
   - qué evidencia o datos se requieren.
3. Stella debe estar disponible como guía contextual, no como chatbot aislado.
4. El usuario debe poder ver progreso.
5. Los riesgos metodológicos deben mostrarse como alertas útiles, no como regaños.
6. Los cálculos deben ser transparentes.
7. Las fuentes deben ser visibles.
8. La trazabilidad debe ser accesible sin saturar la interfaz.
9. Los estados vacíos deben orientar al usuario.
10. Los errores deben ser claros y accionables.

## Estilo de interfaz

Recomendado:
- layout tipo SaaS B2B;
- sidebar izquierda;
- topbar con organización/proyecto;
- cards ejecutivas;
- tablas limpias;
- formularios por pasos;
- panel lateral de Stella;
- badges de estado;
- indicadores de readiness;
- timeline de audit trail;
- vistas comparativas para versiones.

## Componentes clave

- App shell.
- Sidebar.
- Topbar.
- Breadcrumbs.
- Project switcher.
- Organization switcher.
- Stepper del SROI Pipeline.
- Cards de estado.
- Data tables.
- Upload zones.
- Evidence cards.
- Proxy cards.
- Formula panels.
- Audit timeline.
- Report preview.
- Stella assistant panel.
- Alerts metodológicas.
- Empty states.
- Loading states.
- Error states.
- Permission denied state.

## Stella en la interfaz

Stella debe estar integrada en cada paso como:

- panel lateral;
- guía contextual;
- revisor metodológico;
- compositor de texto.

Stella no debe dominar la interfaz. Debe acompañar el trabajo, no reemplazarlo.

Cada interacción con Stella debe dejar claro si su respuesta es:
- sugerencia;
- advertencia;
- explicación;
- borrador;
- validación preliminar.

## Copywriting

Usar lenguaje claro, riguroso y ejecutivo.

Evitar:
- “certificamos tu impacto”;
- “impacto garantizado”;
- “auditoría automática”;
- “resultado definitivo”;
- “la IA comprobó”.

Preferir:
- “audit-ready”;
- “trazable”;
- “defendible”;
- “revisable”;
- “fuente verificable”;
- “supuesto metodológico”;
- “evidencia asociada”;
- “riesgo de atribución”;
- “requiere revisión humana”.

## Accesibilidad

Mínimo desde MVP:

- buen contraste;
- navegación por teclado en formularios;
- labels visibles;
- mensajes de error claros;
- tamaños legibles;
- estados no dependientes solo del color;
- estructura semántica.

## Responsive

El producto debe funcionar bien en desktop y tablet. Mobile debe permitir consulta y revisión básica, pero la construcción completa del SROI Pipeline puede priorizar desktop.
